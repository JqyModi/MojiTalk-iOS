# MojiTalk (MOJi说) UI/UX 设计方案 - Pro Max 版

> 本方案由 `ui-ux-pro-max` 工具分析生成，旨在为 MojiTalk 提供顶级的日语沉浸式对话体验。

## 1. 设计风格：液态玻璃 (Liquid Glass)
基于日语学习的特殊性，我们选择 **“液态玻璃”** 风格，强调流动性、平滑过渡和半透明层次感。

### 视觉特征
- **流动的背景 (Mesh Gradient)**: 采用动态生成的网格渐变，颜色以深色调为主（适配日语沉浸感）。
- **毛玻璃容器 (Animated Blur)**: 对话气泡和控制面板使用 `Ultra Thin Material`，并带有动态模糊效果。
- **高阶阴影 (Soft Elevation)**: 使用极小扩展度、大模糊值的阴影，模拟现实物体的悬浮感。

## 3. 页面布局与交互 (Immersive Experience)

### 3.1 比例分配 (Layout Pattern)
- **Top 45% (Live2D Character)**: 全屏角色交互区。背景采用微动模糊。
    - **交互动画**: 角色在说话时，背景产生微弱的色度偏移 (Chromatic Aberration)。
- **Bottom 55% (Chat Stream)**: 对话流区域。
    - **输入栏**: 悬浮胶囊设计，位于屏幕底部 16pt 处，不贴边。

### 3.2 核心微交互 (Micro-interactions)
- **气泡弹出**: 采用 `.spring(response: 0.4, dampingFraction: 0.6, blendDuration: 0)` 动效。
- **流式增量更新**: AI 文字生成时，字符逐个平滑浮出，带有关联透明度变化，而非生硬跳出。
- **录音反馈**: 长按时，角色底部产生波纹效果，且波纹频率与声音分贝 (Decibel) 绑定。

## 4. SwiftUI 实现准则 (Stack Implementation)
- **动画控制**: 优先使用 `withAnimation(.spring())` 驱动状态变更，避免硬跳转。
- **材质模拟**: 充分利用 SwiftUI 的 `.background(.ultraThinMaterial)` 实现真实的液态玻璃质感。
- **性能优化**: 
    - 使用 `LazyVStack` 处理长对话。
    - 角色容器使用 `UIViewRepresentable` 桥接时，注意控制渲染频率。
